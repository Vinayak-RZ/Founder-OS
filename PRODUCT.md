# Product

## Register
product

## Users & purpose
Solo founders and small-team operators who run Founder OS as an autonomous chief-of-staff. They open the web UI daily to chat with the agent, approve risky actions, scan CRM and runway, and review what the system did overnight. Primary job: stay oriented and in control without living in Telegram.

## Brand personality
Capable, calm, direct. A serious operator console — not a toy chatbot, not enterprise bloat. Feels like a well-built internal tool from a company that ships.

## Visual direction
Ferrari design system via [getdesign](https://getdesign.md/ferrari/design-md): cinematic black canvas (#181818), Rosso Corsa accents (#da291c), sharp corners, uppercase CTAs, Inter as type substitute. See root `DESIGN.md`.

## Anti-references
- Generic AI SaaS (purple gradients, glass cards, hero metrics)
- Rounded pill buttons and soft card grids
- Over-decorated dashboards that hide the actual state
- Chat-only UIs with no operational visibility

## Strategic principles
1. **Task-first** — Chat is primary, but business state is always one click away.
2. **Trust through density** — Show real numbers, pending approvals, and recent actions without fluff.
3. **Earned familiarity** — Standard app shell (sidebar + content), consistent controls.
4. **Accessible by default** — WCAG AA contrast, keyboard nav, reduced-motion support.

## Accessibility
WCAG 2.1 AA minimum. Respect `prefers-reduced-motion`. Focus rings on all interactive elements.
